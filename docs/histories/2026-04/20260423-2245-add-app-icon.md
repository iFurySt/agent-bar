### User Query

> 用户提供了 AgentBar logo 图片，希望用于 DMG 里的 `AgentBar.app`。

### Changes Overview

**Scope:** macOS app packaging

**Key Actions:**

- **[App Icon Asset]**: 从用户提供的 1024 图标母版生成标准 `AgentBar.icns`，在传统 `.icns` 输出中预渲染白色圆角底板，把可见底板缩进到约 80% 画布并把主体放入安全区，作为仓库内版本化资源。
- **[DMG Builder]**: 打包 `AgentBar.app` 时把 icon 复制到 `Contents/Resources`，并在 `Info.plist` 写入 `CFBundleIconFile`。

### Design Intent (Why)

macOS Finder 和 DMG 展示应用图标依赖 app bundle 顶层 `Contents/Resources` 里的 `.icns` 以及 `Info.plist` 的 `CFBundleIconFile`。SwiftPM resource bundle 适合运行时资源加载，但不能单独替代应用 bundle icon 声明。当前手写 `.app` 打包没有 Xcode asset catalog 的系统 mask 链路，透明 logo 直接封装会让 Finder 按非透明内容外接框预览，导致图标过大且保留直角主体边缘；如果圆角底板占满整个 1024 画布，Quick Look 和 Dock 里也会显得比常规 macOS app icon 更大。因此传统 `.icns` 需要输出已经预渲染好的最终 app icon 像素，并保留透明外边距来控制视觉重量。
