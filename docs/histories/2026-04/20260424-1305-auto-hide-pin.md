## [2026-04-24 13:05] | Task: 为外接屏 AgentBar 添加 auto-hide 和 pin

### Execution Context

- **Agent ID**: `Codex`
- **Base Model**: `GPT-5`
- **Runtime**: `Codex CLI`

### User Query

> 用户希望无 notch 的普通屏幕 AgentBar 能在未 pin 时自动收起；当系统顶部菜单栏自动隐藏或全屏应用隐藏菜单栏时，AgentBar 的收起和展开节奏应尽量对齐菜单栏。bar 可见时右侧常驻显示 pin icon，状态需要持久化到下次启动；有 notch 的内建屏幕不需要 pin。

### Changes Overview

**Scope:** macOS AppKit overlay

**Key Actions:**

- **[Auto Hide]**: 为无 notch 的普通屏幕启用自动收起；未 pin 且鼠标离开时把 panel 滑到屏幕顶部外，鼠标进入顶部唤出区或 island 区域时再滑出。有 notch 的屏幕继续不显示 pin。
- **[Pin Preference]**: 在 bar 可见时常驻显示右侧 SF Symbols pin button，点亮代表固定显示；pin 状态写入 `UserDefaults` 的 `AgentBar.pinnedOpen`。
- **[Mouse Handling]**: 保持窗口默认透传鼠标事件，只在可自动隐藏 island 的 hover/pin 操作期间临时接管鼠标，降低挡住系统菜单栏的概率。
- **[Docs]**: 同步 README、架构、界面、稳定性、质量评分和用户可见发布记录。

### Design Intent (Why)

外接屏没有 notch 可作为视觉锚点，且用户可能开启菜单栏自动隐藏或在单屏全屏应用里隐藏顶部菜单栏。如果 AgentBar 始终固定在顶部，会比系统菜单栏更突兀；但完全依赖系统菜单栏可见性又无法覆盖“菜单栏不自动隐藏但用户希望 AgentBar 自己收起”的场景。因此 auto-hide 以 pin 偏好为主，hover 唤出以屏幕顶部区域为触发，并使用接近菜单栏的 0.26s ease-in-out 动画保持观感一致。
